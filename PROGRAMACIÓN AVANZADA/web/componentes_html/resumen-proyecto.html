Resumen del Proyecto Integrador

En Ecuador, el sistema penitenciario enfrenta diversos desafíos, desde problemas de infraestructura hasta gestión de datos. Recientemente, se ha llevado a cabo un proyecto para mejorar la recolección y organización de datos relacionados con las cárceles del país. El proyecto comenzó con la recolección de datos de múltiples fuentes, abarcando el período de 2016 a 2024. Utilizamos csvkit para convertir hojas de Excel a formato CSV, facilitando su manejo. Además, desarrollamos scripts de partición específicos para los datos provenientes de diferentes años, asegurando una organización eficiente y accesible. Este esfuerzo no solo busca mejorar la gestión de datos, sino también proporcionar información más clara y precisa sobre la situación penitenciaria en Ecuador, permitiendo decisiones más informadas y efectivas para la mejora del sistema carcelario.

Resumiendo lo realizado

El presente proyecto se ha centrado en unificar y analizar exhaustivamente múltiples fuentes de datos sobre detenciones desde 2016 hasta 2024. A través de la aplicacion de diversas herramientas y tecnologias, se logro convertir, normalizar y unificar datos procedentes de distintos años y formatos en nuestra estrucutra coherente y analizable. .

Normalización de Esquemas

Un desafío significativo fue la variación en los formatos de datos a lo largo de los años. Para abordar esto, creamos esquemas personalizados que garantizaron la consistencia en las columnas de datos. Este proceso fue crucial para unificar la información y hacerla comparable a lo largo del tiempo.

Procesamiento Avanzado de Datos:

Empleamos una combinación de tecnologías para el procesamiento de datos: Spark y Scala: Utilizados para el manejo inicial de grandes volúmenes de datos. Python con Pandas: Implementado para datos más recientes, permitiendo una limpieza y transformación más detallada. Esta combinación nos permitió adaptar nuestro enfoque a las características específicas de cada conjunto de datos.

Optimización del Almacenamiento:

Adoptamos el formato Parquet para el almacenamiento de datos unificados. Esta decisión mejoró significativamente la eficiencia en el almacenamiento y aceleró el acceso a los datos, crucial para análisis posteriores.

Diseño de la Base de Datos:

El diseño de la base de datos se realizó en dos etapas clave: - Modelo Conceptual: Desarrollamos un Modelo Entidad-Relación (E-R) detallado. - Modelo Lógico: Transformamos el modelo E-R en un Modelo Relacional, optimizado para su implementación en MySQL. Implementación de la Base de Datos: Generamos un script SQL completo para crear la estructura de la base de datos en MySQL. Este script incluyó la definición de todas las tablas, relaciones y restricciones necesarias para mantener la integridad de los datos. Carga de Datos: La carga de datos se realizó en dos fases: - Datos Base: Importamos la información principal sobre detenciones y detenidos. - Datos Complementarios: Incorporamos variables socioeconómicas adicionales para enriquecer el análisis.

Preparación para Análisis:

Como paso final, generamos un archivo CSV consolidado que incluye todas las variables relevantes. Este archivo está diseñado para facilitar análisis futuros y la integración con otras herramientas de análisis de datos.

Desafíos y Soluciones:

Inconsistencia de Datos Superamos las variaciones en los formatos de datos a lo largo de los años mediante un proceso riguroso de normalización. Volumen de Datos Utilizamos tecnologías como Spark y Parquet para manejar eficientemente grandes cantidades de información. Integración de Fuentes Diversas Desarrollamos métodos personalizados para unificar datos de múltiples fuentes y años en un esquema coherente.
